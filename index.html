1NF First Normal Form
▼

Rules

  • Each column holds atomic (indivisible) values
  • No repeating groups or arrays in a column
  • Each row is uniquely identifiable (primary key)
  • All values in a column are of the same type

Violation Example

Orders(OrderID, CustomerName, Products)
→ "Products" stores "Pen, Book, Ruler" — NOT atomic

Fix

Orders(OrderID, CustomerName)
OrderItems(OrderID, ProductID, Quantity)
2NF Second Normal Form
▼

Prerequisite

  • Must already be in 1NF

Rules

  • No partial dependencies — every non-key attribute must depend on the entire composite primary key, not just part of it
  • Only relevant when the primary key is composite

Violation Example

OrderItems(OrderID, ProductID, Qty, ProductName)
→ ProductName depends only on ProductID, not the full key

Fix

OrderItems(OrderID, ProductID, Qty)
Products(ProductID, ProductName, Price)
3NF Third Normal Form
▼

Prerequisite

  • Must already be in 2NF

Rules

  • No transitive dependencies — non-key attributes must not depend on other non-key attributes
  • Every attribute must depend directly on the primary key

Violation Example

Customers(CustomerID, Name, ZipCode, City)
→ City depends on ZipCode, not CustomerID directly

Fix

Customers(CustomerID, Name, ZipCode)
ZipCodes(ZipCode, City, State)

Summary Chain

1NF → atomic values
2NF → no partial deps
3NF → no transitive deps
